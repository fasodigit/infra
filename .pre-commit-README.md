# Pre-commit — FASO DIGITALISATION

## Setup (obligatoire pour chaque développeur)

```bash
# 1. Installer pre-commit
pip install pre-commit

# 2. Installer les dépendances Rust supplémentaires
cargo install cargo-deny cargo-audit --locked

# 3. Installer hadolint (lint Containerfile)
# Linux :
curl -sSfL https://github.com/hadolint/hadolint/releases/latest/download/hadolint-Linux-x86_64 \
  -o ~/.local/bin/hadolint && chmod +x ~/.local/bin/hadolint

# 4. Activer les hooks git dans le repo local
pre-commit install --install-hooks

# Vérification immédiate sur tous les fichiers
pre-commit run --all-files
```

Les hooks s'activent automatiquement sur :

| Stage | Hooks déclenchés |
|-------|-----------------|
| `pre-commit` | file hygiene, gitleaks staged, fmt, clippy, spdx, prettier, shellcheck, hadolint, yamllint |
| `commit-msg` | commitlint (conventional commits) |
| `pre-push` | gitleaks commit scan, cargo deny, cargo audit |

## Types de commit autorisés

`feat` `fix` `docs` `style` `refactor` `perf` `test` `build` `ci` `chore` `revert` `sec`

Exemple : `feat(kaya): add WAL compaction threshold config`

## Bypass — politique stricte

> **Le bypass (`--no-verify`) est interdit sauf urgence documentée.**

Si un hook bloque un commit urgent en production :

1. Ouvrir un ticket `sec` ou `chore` décrivant l'urgence.
2. Notifier `#faso-sec` sur Slack.
3. Utiliser `--no-verify` **une seule fois** en mentionnant le ticket dans le message de commit.
4. Créer un commit de suivi correctif dans les 24 h.

Tout bypass non documenté est considéré comme une violation de la politique de sécurité FASO DIGITALISATION.

## CI/CD

Le workflow `.github/workflows/pre-commit-ci.yml` ré-exécute tous les hooks sur chaque PR.
Un échec bloque le merge même si le développeur n'a pas installé pre-commit localement.
